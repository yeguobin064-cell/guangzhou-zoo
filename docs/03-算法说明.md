# 03 - 算法说明

## 1. Dijkstra 最短路径算法

**用途**：菜单 4（查询最短路径）、菜单 5（规划路线）、菜单 8（所有路径）

**原理**：从起点出发，每次选取未访问顶点中距离最小的一个，更新其邻接顶点的距离，直到到达终点。

**实现**：
- 距离数组 `dist[]`，前驱数组 `prev[]`，访问标记 `visited[]`
- 时间复杂度：O(V^2)，V 为顶点数
- 空间复杂度：O(V)

**输出**：最短路径序列、总距离、步行时间、途经景点数。

## 2. Floyd 全源最短路径算法

**用途**：计算所有顶点对之间的最短距离（用于内部计算）

**原理**：三重循环，枚举中间节点 k，若 dist[i][k] + dist[k][j] < dist[i][j] 则更新。

**实现**：
- 距离矩阵 `dist[MAX_VERTEX_NUM][MAX_VERTEX_NUM]`
- 路径矩阵 `path[MAX_VERTEX_NUM][MAX_VERTEX_NUM]`
- 时间复杂度：O(V^3)
- 空间复杂度：O(V^2)

## 3. DFS 导游线路算法

**用途**：菜单 6（DFS导游线路）

**原理**：从指定起点出发，使用栈实现深度优先遍历，访问所有可达景点。

**实现**：
- 使用显式栈（数组模拟）
- 访问标记数组 `visited[]`
- 邻接点逆序压栈保证遍历顺序
- 时间复杂度：O(V + E)

## 4. 回路检测算法

**用途**：菜单 7（检测回路）

**原理**：基于 DFS 的递归栈检测。若在遍历过程中遇到已在递归栈中的节点，则存在回路。

**实现**：
- 递归栈标记 `recStack[]`
- 父节点记录 `parent[]` 用于还原回路路径
- 时间复杂度：O(V + E)

## 5. 贪心游览路线规划算法

**用途**：菜单 5（规划游览路线）

**原理**：从起始景点出发，每次选择最近的未访问景点，直到时间用尽或无可达景点。

**实现**：
- 从当前顶点的邻接表中找最近未访问顶点
- 预估时间 = 已用时间 + 行走时间 + 目标景点游览时间
- 若超过时间限制则跳过该候选
- 时间复杂度：O(V^2)

## 6. 随机路线生成算法（含公共厕所）

**用途**：菜单 13

**算法步骤**：

1. **定位入口**：找到 TYPE_ENTRANCE 顶点作为起点
2. **收集厕所**：找到所有名称含"厕所"的 TYPE_FACILITY 顶点
3. **候选景点**：排除入口和厕所，收集剩余景点
4. **Fisher-Yates 洗牌**：随机打乱候选景点顺序
5. **选取景点**：从洗牌结果中取前 3~5 个
6. **智能插入厕所**：在路线约 1/3 处，选择离当前景点最近的公共厕所插入
7. **Dijkstra 连接**：对相邻途经点调用 Dijkstra 求最短路径，拼接为完整路径
8. **输出路线**：显示完整路径、每段距离时间、总距离时间、厕所位置

**随机性保证**：
- 使用 `time(NULL)` 作为随机种子
- Fisher-Yates 洗牌保证均匀随机

**约束保证**：
- 路线必须从入口出发
- 路线必须包含至少一个公共厕所
- 厕所选择策略：距离最近优先
