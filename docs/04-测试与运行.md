# 04 - 测试与运行

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10+ / Linux / macOS |
| 编译器 | GCC 11+ / Clang 13+ / MSVC 2019+ |
| 构建工具 | CMake 3.16+ / Ninja |
| 终端 | 支持 ANSI 转义码（Windows 10 及以上） |

## 2. 编译与运行

```bash
# 进入项目目录
cd D:\SAPPS\guangzhou-zoo

# 创建构建目录（首次）
mkdir build && cd build
cmake .. -G Ninja

# 编译
ninja

# 运行
.\bin\zoo_route.exe
```

## 3. 功能测试用例

### 测试 1：查看景点列表（菜单 2）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 2 |
| 预期 | 显示 10 个景点，含名称、类型、游览时间 |

### 测试 2：查询最短路径（菜单 4）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 4，起点选 1（正门），终点选 4（海洋馆） |
| 预期 | 输出路径：正门 -> 大象馆 -> 海洋馆，距离 550 米 |

### 测试 3：DFS 导游线路（菜单 6）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 6，起点选 1（正门） |
| 预期 | 输出从正门出发的深度优先遍历路线，经过所有可达景点 |

### 测试 4：添加景点（菜单 9）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 9，名称输入"爬行馆"，类型选 1，时间 20，人气 6 |
| 预期 | 提示"添加景点成功"，景点列表变为 11 个 |

### 测试 5：删除景点（菜单 10）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 10，选刚才添加的景点编号，确认删除 |
| 预期 | 提示"删除景点成功"，景点列表恢复 10 个 |

### 测试 6：随机路线（菜单 13）

| 项目 | 内容 |
|------|------|
| 操作 | 输入 13，时间限制输入 120 |
| 预期 | 输出从正门出发的随机路线，包含 3~5 个景点和至少 1 个公共厕所，显示距离和时间 |

### 测试 7：错误输入处理

| 项目 | 内容 |
|------|------|
| 操作 | 菜单选择时输入 99 或 -1 |
| 预期 | 提示"输入无效"，重新要求输入 |

## 4. 异常场景测试

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 添加重名景点 | 菜单 9 添加已存在的名称 | 提示"景点名称已存在" |
| 查询不存在的景点 | 菜单 11 按名称查询"不存在" | 提示"未找到该景点" |
| 删除景点确认 | 菜单 10 选择取消 | 提示"已取消删除" |
| 不可达路径 | 菜单 4 查询断开的两个景点 | 提示"无法找到路径" |

## 5. 项目文件说明

| 文件 | 作用 |
|------|------|
| `main.c` | 主程序入口，菜单循环 |
| `graph.c` | 图操作：创建、遍历、增删改查 |
| `pathfinding.c` | 算法实现：Dijkstra、Floyd、DFS、随机路线 |
| `zoo_data.c` | 广州动物园初始化数据 |
| `consoleio.c` | 终端界面：颜色、输入输出 |
| `zoo_layout.txt` | 景点布局数据文件 |
